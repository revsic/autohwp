## Writing style guide

문체
- 갠관적, 공식적 문체
- 기술 설명 + 정책적 필요성 병행
- 연구 계획 중심 서술
- 결과 약속형 문장 구조
- 1인칭이 아닌 3인칭 객관 서술
- 불확실성 대신 단정적 표현을 담백히 서술
- 문제 정의, 기존 한계, 제안 방법, 기대 효과의 설득 구조
- 기술, 산업, 국가 인프라로의 구조적 확장 연결 (e.g., GPU Efficiency, AI research ecosystem, Physical AI)
- 기술명은 영문을 유지한다.

예시
- 본 과제는 … 문제를 해결하기 위한 … 방법론을 제안한다
- 본 연구는 … 기반으로 수행된다
- 이를 통해 … 성능을 확보하는 것을 목표로 한다

## 1. 전체 스타일 개요

### 1.1 핵심 특징

- **목표 지향적 서술**: 모든 문단이 “문제 → 해결 → 기여” 구조를 따른다
- **문장 길이: 중~장문 위주**: 한 문장에 2~4개의 정보 단위 포함
- **비문학적이지만 설명 중심**: 주장보다는 “설명 + 정당화” 중심
- **용어 밀도 높음**: consistency, reproducibility, instability 등 핵심 키워드 반복
- **감정/수사 없음**: 철저히 기능적 문장

## 2. 문체 (Tone & Syntax)

### 2.1 기본 어투

- 평서형, 단정형
- "~한다", "~이다" 중심
- 추측 표현 거의 없음 (e.g., “~일 수 있다” 최소화)

### 2.2 문장 구조 패턴

#### (1) 원인-결과 구조

> “~로 인해 ~ 문제가 발생한다”

예시:

> 실제 학습 과정에서 불안정성 모델 간 성능 편차 재현성 부족 등 핵심적인 문제가 발생한다 

#### (2) 문제-해결 구조

> “~ 문제를 해결하기 위해 ~을 제안한다”

예시:

> 본 과제는 이러한 불안정성을 해소하기 위해 자기 증류 기법을 활용한 방법을 제안한다 

#### (3) 목적 명시 구조

> “~을 목표로 한다”

예시:

> 학습의 불안정성과 재현성 문제를 해소하는 것을 목표로 한다 

#### (4) 병렬 나열 구조

- 쉼표 기반 나열 (and 최소 사용)

예시:

> 학습 불안정성, 모델 간 성능 편차, 재현성 부족 등

### 2.3 특징적인 문법

- 조사 생략 빈번 (특히 “은/는”, “이/가”)
- 접속사 최소화
- 문장 간 연결은 **의미적 연결**로 처리

## 3. 구조 (Document Organization)

### 3.1 기본 전개 구조

모든 섹션이 아래 패턴을 반복:

```
[문제 정의]
→ [기존 한계]
→ [제안 방법]
→ [기대 효과]
```

### 3.2 실제 예시 구조

#### (1) 문제 정의

> 확산형 생성모델은 높은 품질을 제공하지만 추론 지연이 존재한다 

#### (2) 기존 한계

> 동일 설정에서도 성능이 크게 달라지는 문제가 존재한다 

#### (3) 제안

> 자기 증류 기반 학습 방법을 제안한다 

#### (4) 효과

> 재현성과 안정성을 동시에 개선한다 

## 4. 논리 전개 방식

### 4.1 핵심 전략: "압축된 논리"

- 한 문단 안에서:

  - 문제 + 원인 + 영향 + 해결 방향을 동시에 설명

### 4.2 논리 유형

#### (1) 구조적 문제 정의

- “구조적 한계”라는 표현 반복 사용

> 구조적 한계 / 구조적 불안정성 / 구조적 병목

#### (2) 정량적 정당화

- 항상 숫자/지표 포함

예시:

- FID 10 이하
- 표준편차 0.21 이하

→ **주장 = 항상 measurable**

#### (3) 비교 기반 논리

- “기존 vs 제안” implicit 비교

> 기존 연구 대비 / 동일 환경에서 / 평균 대비

#### (4) 범용성 확장

- 항상 마지막에 확장성 언급

> 이미지 → 음성 → 로봇 → 단백질

## 5. 표현 패턴 (Reusable Templates)

### 5.1 문제 정의 템플릿

```
최근 ~은 ~의 장점을 가지나, ~라는 한계를 가진다
```

### 5.2 문제 심화 템플릿

```
특히 ~로 인해 동일한 설정에서도 ~ 문제가 발생한다
```

### 5.3 제안 템플릿

```
본 연구는 ~을 통해 ~ 문제를 해결하고자 한다
```

### 5.4 목표 템플릿

```
정량적으로는 ~을 달성하는 것을 목표로 한다
```

### 5.5 기여 템플릿

```
이를 통해 ~을 동시에 확보한다
```

## 6. 분량 감각

### 6.1 문단 길이

- 평균: 5~8 문장
- 각 문단 = 하나의 완결된 논리 블록

### 6.2 문장 길이

- 평균 30~60자 (한국어 기준으로는 꽤 긴 편)
- 정보 밀도 높음 (불필요한 수식어 없음)

### 6.3 정보 밀도

한 문장에 포함되는 요소:

- 문제 + 원인 + 영향
  or
- 방법 + 특징 + 효과

## 7. 키워드 사용 전략

### 7.1 반복 키워드

- 재현성 (reproducibility)
- 안정성 (stability)
- 불안정성 (instability)
- 구조적 (structural)
- 저지연 (low-latency)

→ 같은 단어 반복 사용 (동의어 회피)

### 7.2 기술 용어 사용 방식

- 정의 없이 바로 사용
- 대신 문맥으로 의미 전달

## 8. Writing DNA (핵심 요약)

이 문체는 한 줄로 정리하면:

> **“문제를 구조적으로 정의하고, 정량적으로 설득하며, 해결책을 기능적으로 제시하는 설명형 연구 문체”**

## 9. 스타일 재현 규칙

이 스타일을 그대로 쓰려면:

1. 문장 시작은 항상 “최근 ~”, “본 연구는 ~”로 잡는다
2. 문제를 “구조적 한계”로 정의한다
3. 반드시 **재현성 / 안정성 / 효율성 중 1개 이상 포함**
4. 모든 주장에는 원인 or 결과 or 수치 중 하나를 붙인다
5. 마지막 문장은 항상 “확장 가능성” 또는 “기여”로 끝낸다

## 10. 간단한 before/after 예시

### ❌ 일반적인 문장

> 이 방법은 성능이 좋다

### ✅ 스타일

> 본 방법은 동일 조건에서 반복 실험 시 성능 편차를 감소시키며 안정적인 수렴 특성을 보인다
